# Thread Scaffolds by Archetype

> **Use:** when generating a thread (medium or long) in the 10 example posts, pick the scaffold for the matched archetype and fill in with founder-specific substance. Do not use the labels ("T1:", "T2:", etc.) in the final output — they're internal structure, not part of the post.

## Protocol Philosopher — Tradeoff Unpacking Thread

```
T1  [Counterintuitive claim. One sentence. e.g. "X is a feature, not a bug."]
T2  [The intuitive view most people hold about X.]
T3  [What that intuitive view misses — the crucial context.]
T4  [The real constraint or reason the design is correct.]
T5  [Optional — implication for current or future work.]
```

## Ecosystem Operator — FUD Rebuttal Thread

```
T1  [Direct opener addressing the FUD head-on. Screenshot or quote optional.]
T2  [Point 1 refutation. With receipts — specific data, chart, or quote.]
T3  [Point 2 refutation. Receipts.]
T4  [Point 3 refutation. Receipts.]
T5  [Optional — the pattern behind why this FUD keeps coming up.]
T6  [Restate the real picture. Invite the audience to judge by data not vibes.]
```

## Ecosystem Operator — Translator Thread

```
T1  [Hook: "A lot of people are confused about X. Here's what's actually happening."]
T2  [Concept 1 broken down simply.]
T3  [Concept 2 broken down simply.]
T4  [Concept 3 broken down simply. How the concepts fit together.]
T5  [What the audience should do next — try it, fund it, build on it.]
```

## Engineer Minimalist — (rare) Launch Thread

```
T1  [What shipped. One line.]
T2  [Why it matters. One line.]
T3  [Optional — specific technical detail worth surfacing.]
T4  [How to try it or where to see it.]
```
Use sparingly. Most Engineer Minimalist posts are not threads.

## Degen Builder Poet — Launch Announcement

```
T1  [Degen-native hook. Not marketing-speak. Something like "the thing you've all been asking for" or a one-line riff on the product category.]
T2  [Product beat 1 with personality.]
T3  [Product beat 2 with personality — something a tiny bit memey or culture-aware.]
T4  [Optional — beat 3.]
T5  [CTA in register — "wen" jokes, specific callouts, or a direct try-it link.]
```

## Contrarian Trader — Thesis Thread

```
T1  [Hook: specific narrative or positioning call.]
T2  [Evidence 1 — market-structure observation or data point.]
T3  [Evidence 2 — historical analog, flow, or liquidity observation.]
T4  [Specific expectation or prediction, often with a time horizon.]
T5  [Optional — what would disprove the thesis.]
```

## Research Maximalist — Framework Thread

```
T1  [Hook — counterintuitive or reframing claim.]
T2  [Definitions / setup of the framework you'll use.]
T3  [Application 1 — example or case study.]
T4  [Application 2 — example or case study.]
T5  [Application 3 — example or case study.]
T6  [Pattern across the applications.]
T7  [Implications and open questions.]
T8  [The bigger claim this supports. Link to long-form write-up if any.]
```

## Research Maximalist — Numbered-Argument Thread

```
T1  [Claim, stated plainly.]
T2  [1. First supporting point with evidence.]
T3  [2. Second supporting point with evidence.]
T4  [3. Third supporting point with evidence.]
T5  [4. Optional — fourth point.]
T6  [5. Optional — fifth point.]
T7  [Restate the claim now that the case is made.]
```

## Consumer Storyteller — Release Narrative Thread

```
T1  [What we just shipped, in human terms — not feature-list.]
T2  [Who it's for and why we built it this way. One concrete user in mind.]
T3  [A user moment or conversation that shaped the design.]
T4  [What's next — what you're trying to build toward.]
T5  [Gratitude + call to try it. Without being saccharine.]
```

## Memetic Provocateur — (rare) Narrative-Setting Mini-Thread

```
T1  [The observation or cultural moment being named.]
T2  [The implication — said sideways or dryly.]
T3  [Optional — the serious point underneath the frame.]
```
Rare. Most Memetic Provocateur posts are single-tweet.

---

## General thread hygiene (applies to all)

- **Tweet 1 does the work.** Should stand alone. If someone sees only T1, they should understand the thread's premise.
- **No "thread 🧵" preamble** unless the founder's real voice uses it (some do, most don't).
- **No "continued in next tweet" signposting.** Let the thread speak for itself.
- **Vary tweet lengths within the thread.** Monotonous length = algorithmic feel.
- **Last tweet lands.** Don't end with "let me know your thoughts!" — end with the thesis, a specific call, or a stark observation. Generic CTAs kill threads.
